1. .Net 알려진 문제

1.1 NetFx 설치

1.1.1 Windows 8에서 .NET Framework 4.5 언어 팩을 사용할 수 없음

.NET Framework 4.5는 운영 체제의 구성 요소이기 때문에 .NET Framework 4.5 언어 팩을 Windows 8 운영 체제에 설치할 수 없습니다. 지원되는 이전 버전의 Windows에 .NET Framework 4.5 언어 팩을 설치할 수 있습니다.

이 문제를 해결하려면

Windows 8 운영 체제용 언어 팩을 다운로드하거나, 지역화된 버전의 Windows 8을 설치하여 .NET Framework 4.5용 지역화된 리소스를 가져옵니다.

1.1.2 .NET Framework 4.5 설치에 실패하거나 설치를 취소하면 응용 프로그램 풀이 2.0으로 되돌려짐

.NET Framework 4에서 .NET Framework 4.5 RTM 릴리스로 업그레이드하는 경우 설치가 취소되거나 실패하면 .NET Framework가 버전 4로 되돌려집니다. 그러나 ASP.NET 4도 등록 취소되고 IIS의 모든 응용 프로그램 풀이 .NET Framework 2.0을 대상으로 하도록 설정됩니다.

이 경우 ASP.NET 4 기반 웹 응용 프로그램을 검색하면 "compilation" 요소의 "targetFramework" 특성을 인식할 수 없다고 보고하는 구성 오류가 발생합니다.

이 문제를 해결하려면

이 문제를 해결하려면

  1. 아래의 옵션 A나 옵션 B를 완료하여 잘못된 등록을 수정합니다(A와 B를 둘 다 완료할 필요는 없음).
    • 다음 명령을 사용해 ASP.NET IIS 등록 도구(aspnet_regiis)를 실행하여 ASP.NET 4를 다시 등록합니다.
      • %windir%\Microsoft.Net\Framework(64)\v4.0.30319\aspnet_regiis.exe -iru -enable

      -또는-

    • 설치 프로그램을 다시 실행하고 복구 옵션을 선택하여 .NET Framework 4 Extended Profile을 복구합니다.

  2. 또한 업그레이드가 실패하기 전에 ASP.NET 4를 대상으로 하는 사용자 지정 응용 프로그램 풀이 있었던 경우 IIS 관리자(inetmgr.exe)를 사용하여 ASP.NET 4를 대상으로 하도록 수동으로 다시 설정합니다.

1.2 NetFx 제거

1.2.1 Windows 8 또는 Windows Server 2012에서 .NET Framework 4.5를 제거한 후에 ASP.NET 2.0 및 3.5가 작동하지 않음

Windows 8 및 Windows Server 2012에서 ASP.NET 2.0 및 3.5를 사용하려면 ASP.NET 4.5 기능을 사용해야 합니다. .NET Framework 4.5를 제거하거나 사용하지 않으면 ASP.NET 2.0 및 3.5 응용 프로그램이 더 이상 실행되지 않습니다.

이 문제를 해결하려면

Windows 8의 경우

제어판에서 ASP.NET 4.5 기능을 사용합니다.

  1. 제어판을 엽니다.
  2. 프로그램을 선택합니다.
  3. 프로그램 및 기능 머리글 아래에서 Windows 기능 사용/사용 안 함을 선택합니다.
  4. .NET Framework 4.5 Advanced Services 노드를 확장합니다.
  5. ASP.NET 4.5 확인란을 선택합니다.
  6. 확인을 선택합니다.

Windows Server 2012의 경우

IIS 학습 센터에서 IIS 8.0 Using ASP.NET 3.5 and ASP.NET 4.5를 참조하십시오.

1.3.8 WCF(Windows Communication Foundation)

1.3.8.1 WCF 4.5에서 기존 XML serialization 코드 실행 시 발생하는 문제

WCF(Windows Communication Foundation) 4.5에서는 XmlSerializer 클래스가 최적화되어 C# 컴파일러에 대한 종속성이 제거되었습니다. 이로 인해 콜드 시작 시나리오에서 성능이 크게 개선됩니다. 하지만 이로 인해 WCF 4에서 컴파일되었지만 WCF 4.5에 대해 실행되고 있는 XML serialization 코드에 문제가 발생할 수도 있습니다.

이 문제를 해결하려면

WCF 4.5에서 기존 XML serialization 코드를 실행할 때 문제가 발생하는 경우 다음 구성 요소를 사용하여 WCF 4에서의 XmlSerializer 동작으로 되돌리십시오.

<configuration>
   <system.xml.serialization>
      <xmlSerializer useLegacySerializerGeneration="true"/>
   </system.xml.serialization>
</configuration>

1.3.10 Windows WF(Workflow Foundation)

1.3.10.1 Workflow Identity 매개 변수가 포함된 워크플로를 사용할 경우 호스트가 시작되지 못함

.NET Framework 4.5에는 SQL Workflow Instance Store에 보관되는 Workflow Identity라는 새로운 워크플로 매개 변수가 있습니다. 이 매개 변수를 포함하는 워크플로를 사용하는데 해당 값이 null이 아닌 경우에는 이 값을 저장할 수 있도록 Instance Store를 업데이트해야 합니다. 그렇지 않으면 호스트가 시작되지 못합니다.

이 문제를 해결하려면

다음 위치에서 스크립트를 실행하여 SQL Workflow Instance Store를 업데이트합니다.

%windir%\Microsoft.NET\Framework\<version>\SQL\<language>\SqlWorkflowInstanceStoreSchemaUpgrade.sql

1.3.10.2 Workflow Designer의 일부 새 기능으로 인해 기존 솔루션에서 문제가 발생할 수 있음

.NET Framework 4.5에서 Workflow Designer는 다음과 같이 변경되었습니다.

  • 이제 여러 개의 작업을 선택했을 때 바로 가기 메뉴를 사용할 수 있습니다.
  • 작업 디자이너에서 이제 전체 디자이너 대신 디자이너의 헤더를 끌어 당길 수 있습니다. FlowDecision 및 FlowSwitch 디자이너에서는 여전히 전체 디자이너를 끌어 당기게 됩니다.
  • Ctrl 키를 누른 상태에서 다중 선택을 할 때의 동작 변경: .NET Framework 4에서는 마우스를 누를 때 선택이 취소됩니다. .NET Framework 4.5 RC에서는 마우스를 눌렀다 놓을 때 선택이 취소됩니다. 마우스를 누른 상태에서는 선택이 그대로 유지됩니다.
  • 이제 작업 디자이너가 작업 대리자를 지원합니다. 자세한 내용은 MSDN 라이브러리의 "방법: Workflow Designer에서 작업 대리자 정의 및 사용"을 참조하십시오.
  • 이제 변수 및 인수 디자이너가 바로 가기 메뉴를 지원합니다. 바로 가기 메뉴에는 변수 및 인수를 삭제하기 위한 항목 그리고 주석 작업을 위한 항목이 있습니다. 자세한 내용은 MSDN 라이브러리의 "방법: Workflow Designer에서 워크플로에 설명 추가"를 참조하십시오.
  • 이전에는 MetadataStore 클래스가 일반 형식 정의의 generic 속성에 특성을 추가하는 것을 지원하지 않았습니다. 이제는 지원됩니다.
  • 이제 ModelItemDictionary 클래스가 null 키를 지원합니다.
  • 이전에는 모델 트리에 없는 항목이 ModelItem.Parents 속성에 포함되었습니다. 이제는 ModelItem.Parents에 있는 모든 개체가 모델 트리에 있는 유효 개체입니다.
  • 이전에는 Switch 작업에 대해 디자이너에서 Case를 선택하면 선택한 Case에 있는 작업으로 키보드 포커스가 이동하고 해당 작업이 강조 표시되었습니다. 이제는 해당 작업이 강조 표시되지만 키보드 포커스는 바뀌지 않습니다.
  • 이전에는 식 텍스트 상자에 한 문자만 입력할 때 디자이너가 값을 Char 형식의 상수로 취급했습니다. 이제는 이 식이 변수 또는 인수 이름으로 취급됩니다.
  • 이전에는 참조되지 않은 어셈블리와 종속 관계가 있는 참조된 어셈블리에 있는 형식이 형식 브라우저에 표시되었습니다. 이제는 종속 어셈블리를 참조하기 전에는 참조되지 않은 어셈블리와 종속 관계가 있는 형식이 나타나지 않습니다.
  • 이제 다음 형식 및 멤버는 더 이상 사용되지 않습니다.
    • System.Activities.Presentation.Services.ModelChangedEventArgs 클래스에 있는 대부분의 속성이 사용되지 않습니다. 사용되지 않는 속성 대신 ModelChangeInfo 속성을 사용하십시오.
    • System.Activities.Presentation.DragDropHelper 클래스에서 새로운 다중 선택 기능에 의해 다음 메서드는 더 이상 사용되지 않습니다.
      • DoDragMove(WorkflowViewElement, Point) - DoDragMove(IEnumerable<WorkflowViewElement>, Point)를 대신 사용하십시오.
      • GetDroppedObject(DependencyObject, DragEventArgs, EditingContext) - GetDroppedObjects(DependencyObject, DragEventArgs, EditingContext)를 대신 사용하십시오.
      • GetDraggedModelItem(DragEventArgs) - GetDraggedModelItems(DragEventArgs)를 대신 사용하십시오.
      • GetCompositeView(DragEventArgs) - GetCompositeView(WorkflowViewElement)를 대신 사용하십시오.
      • SetDragDropCompletedEffects(DragEventArgs, DragDropEffects) - SetDragDropMovedViewElements(DragEventArgs, IEnumerable<WorkflowViewElement>)를 대신 사용하십시오.
      • GetDragDropCompletedEffects(DataObject) - GetDragDropMovedViewElements(DataObject)를 대신 사용하십시오.

이 문제를 해결하려면

2. .Net 유지 관리

2.1 Windows 8로 업그레이드할 때 .NET Framework 4 언어 팩이 업데이트되지 않음

Windows 7 운영 체제에서 Windows 8로 업그레이드하는 경우 이전에 컴퓨터에 설치한 .NET Framework 4 언어 팩이 제거되거나 업데이트되지 않습니다. 이 문제는 영어를 포함하지 않는 업그레이드된 Windows 8의 언어와 일치하지 않는 언어 팩에 영향을 줍니다. 예를 들어 영어 버전의 Windows 7 Ultimate와 .NET Framework 4 독일어 언어 팩이 있는 상태에서 시스템을 영어 버전의 Windows 8로 업그레이드하는 경우 언어 팩이 시스템에 남아 있기는 하지만 .NET Framework 4.5로 업데이트되지 않습니다.

이 문제를 해결하려면

Windows 8로 업그레이드하기 전에 .NET Framework 4 언어 팩을 제거하거나, 업그레이드한 후에 적절한 Windows 8 언어 팩을 설치합니다.
